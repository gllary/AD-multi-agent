Supplementary File 3. Safety-governance runtime settings and audit logs

This plain-text supplementary file preserves detailed runtime, fallback, and safety-governance audit information that was moved out of the Supplementary Information PDF for readability. It does not define new analyses and was not used to tune prompts, thresholds, routing policies, model backends, or safety rules after external-cohort exposure.

Section A. LLM runtime and failure handling

These settings apply to specialist, coordinator, and single-agent calls.

Component: Backend
Frozen setting or handling rule: Qwen3-235B-A22B-Instruct through an OpenAI-compatible local chat-completions interface.

Component: Maximum context length
Frozen setting or handling rule: 8,192 tokens.

Component: Deployment engine
Frozen setting or handling rule: vLLM deployment on a private institutional server.

Component: Hardware
Frozen setting or handling rule: Four 141-GB H20 GPUs.

Component: Third-party API use
Frozen setting or handling rule: None; patient-level LLM calls were not sent to a third-party API provider.

Component: PHI export risk control
Frozen setting or handling rule: De-identified study IDs and stage-bounded evidence packets were processed inside the private server environment, with no protected health information exported outside the institutional deployment.

Component: Temperature
Frozen setting or handling rule: 0.2 for reported validation evaluations.

Component: Prompt specification
Frozen setting or handling rule: Specialist and coordinator prompts frozen before any external-cohort exposure.

Component: Patient identifiers
Frozen setting or handling rule: De-identified study IDs and stage-bounded evidence packets used for analysis.

Component: Output constraint
Frozen setting or handling rule: One JSON object conforming to the supplied schema; allowed action values supplied in the user payload.

Component: Timeout threshold
Frozen setting or handling rule: 120 s per raw API attempt before retry/failure handling.

Component: Retry and resumption
Frozen setting or handling rule: Transient timeout/API/transport failures were retried, and checkpointed outputs allowed interrupted validation evaluations to resume from the last validated call; 3,367 retry events occurred before the final retained outputs were obtained.

Component: Malformed or missing JSON
Frozen setting or handling rule: Deterministic fallback record generated from stage, role, risk score, and risk band.

Component: Illegal action
Frozen setting or handling rule: Rejected by the deterministic safety layer and reverted to the canonical threshold route.

Component: Free-text reasoning
Frozen setting or handling rule: Full chain-of-thought was not requested or used; bounded rationale summaries and evidence lists were stored.
